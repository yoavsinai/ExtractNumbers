# Data Preparation Module

This module handles the fetching, parsing, and normalization of various image datasets for number recognition. It consolidates multiple sources (real-world and synthetic) into a **unified directory structure and metadata format**.

## Unified Data Structure

Processed data is saved in `data/digits_data/` with the following structure:

```text
data/digits_data/
├── <dataset_name>/           (e.g., svhn, race_numbers, handwritten)
│   ├── sample_<id>/          (Unique ID for each image)
│   │   ├── original.png      (The source image)
│   │   └── annotations.json  (Consolidated metadata)
```

## Metadata Schema (`annotations.json`)

Each sample contains a standardized JSON file meticulously designed to support localization and recognition tasks:

```json
{
    "image_metadata": {
        "sample_index": 0,       // 0-based index within the specific dataset
        "filename": "original.png",
        "width": 1024,
        "height": 768
    },
    "detected_numbers": [        // Array supporting multiple numbers per image
        {
            "full_value": "123", // The complete number sequence
            "full_bounding_box": {
                "x": 100.0,      // Top-left X
                "y": 150.0,      // Top-left Y
                "width": 200.0,
                "height": 50.0
            },
            "digits": [          // Optional granular digit data (if available)
                {
                    "label": 1,
                    "bounding_box": { "x": 100.0, "y": 150.0, "width": 60.0, "height": 50.0 }
                },
                ...
            ]
        }
    ]
}
```

## Pipeline Phases

The preparation process is divided into two distinct phases:

1.  **Phase 1: Fetching & Preparation**: Individual loaders in `src/data/` fetch raw data (via `kagglehub`), parse annotations, and organize them into the unified structure.
2.  **Phase 2: High-Level Augmentations**: A global post-processor ([apply_augmentations.py](apply_augmentations.py)) walks through the entire `digits_data` directory and applies randomized degradations to improve model robustness.

### Augmentations Applied:
*   **White Noise**: Gaussian noise with randomized intensity.
*   **Gaussian Blur**: Variable radius blur.
*   **Geometric Stretches**: Randomized horizontal/vertical resizing. **Note**: This phase automatically recalculates all bounding box coordinates in `annotations.json` to ensure they remain perfectly aligned with the stretched images.

## Available Datasets

| Dataset | Source | Description |
| :--- | :--- | :--- |
| **SVHN** | Kaggle (`stanfordu/street-view-house-numbers`) | Street View House Numbers (Format 1). Real-world house numbers. |
| **Race Numbers** | Kaggle (`trainingdatapro/ocr-race-numbers`) | Numbers on marathon runners. Complex real-world backgrounds. |
| **Handwritten** | Synthetic (MNIST-based) | Generated by planting scaled-up MNIST digits into natural images from Kaggle (`prasunroy/natural-images`). |

## Usage

The main entry point for the entire pipeline is the root script:
```bash
python src/prep_data.py
```

### Options:
- `--limit N`: Process only N samples per dataset (useful for testing).
- `--datasets d1 d2 ...`: Choose specific datasets (e.g., `--datasets svhn`).
- `--clean`: Wipe existing `digits_data/` before starting (no longer asks for confirmation).
- `--no-augment`: Skip the Phase 2 high-level augmentation phase entirely.

1.  Create a new loader script in `src/data/` (e.g., `my_new_dataset.py`).
2.  Implement a `prepare(output_base_dir, limit=None)` function.
3.  Ensure it adheres to the **Unified Data Structure** and **Metadata Schema** above.
4.  Import and call it in `src/prep_data.py`.
